PROBLEMA 1 — DRAG CON MOUSE INVERTIDO
==================================================

ACLARACIÓN IMPORTANTE:
El problema YA NO es que el movimiento esté cruzado o rotado 90°.

El problema actual es este:

- si arrastro el mouse hacia abajo, el plano sube
- si arrastro el mouse hacia arriba, el plano baja
- si arrastro el mouse hacia la izquierda, el plano va a la derecha
- si arrastro el mouse hacia la derecha, el plano va a la izquierda

O sea:
el drag del mouse está invertido en ambos ejes.

Esto hay que entenderlo bien:
NO está girado
NO está cruzado
ESTÁ AL REVÉS / ESPEJADO

OBJETIVO:
el plano debe seguir al cursor de forma directa, natural y 1:1

Resultado correcto:
- mouse arriba → plano arriba
- mouse abajo → plano abajo
- mouse izquierda → plano izquierda
- mouse derecha → plano derecha

IMPORTANTE:
- teclado y botones ya funcionan bien
- NO tocar esa lógica
- corregir SOLO la lógica del drag con mouse

REVISAR CONCRETAMENTE:
- cálculo de dx / dy
- si se está usando start - current en vez de current - start
- signos invertidos en X/Y
- si al aplicar el delta final se invirtieron ambos ejes

PRIORIDAD:

PROBLEMA 2 — HANDLE DE ROTACIÓN MAL UBICADO
==================================================

Estado actual:
- el handle de rotación quedó en el centro del plano
- eso es incorrecto

UBICACIÓN CORRECTA:
el handle debe estar SIEMPRE en el centro del borde superior del plano

Es decir:
- entre la esquina superior izquierda y la esquina superior derecha
- exactamente en el punto medio entre ambas
- con un offset visual hacia afuera

IMPORTANTE:
- NO usar el centroide del plano
- NO usar el centro general del bounding
- usar las dos esquinas superiores reales del plano

Cálculo esperado:
1. tomar topLeft y topRight
2. calcular midpoint(topLeft, topRight)
3. aplicar un offset perpendicular hacia afuera del borde superior
4. renderizar ahí el handle

COMPORTAMIENTO ESPERADO:
- si el plano rota, el handle debe seguir estando arriba
- si el plano se mueve, el handle se mueve con él
- si el plano gira, el handle NO debe terminar en el medio ni orbitar raro
