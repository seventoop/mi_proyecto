ALCANCE DE LA AUDITORÍA

Quiero una auditoría general, no una revisión línea por línea de todo el repositorio.
Priorizá:
- estructura del proyecto
- archivos y módulos más importantes
- configuración general
- dependencias
- calidad de integración entre frontend / backend / servicios
- lugares donde se vea fragilidad técnica
- patrones repetidos o inconsistentes
- señales de código obsoleto, duplicado o no utilizado
- posibles problemas de tipos, warnings, efectos colaterales, acoplamiento o lógica difícil de sostener

TAMBIÉN QUIERO QUE EVALUES

SEGURIDAD
- exposición de secretos, tokens o credenciales
- uso inseguro de variables de entorno
- validaciones débiles
- riesgos de autenticación/autorización
- endpoints sensibles
- posibles problemas de permisos
- posibles riesgos XSS, CSRF, inyección o manejo inseguro de inputs, si aplica
- cualquier práctica que pueda traer problemas en producción

OPTIMIZACIÓN / PERFORMANCE
- componentes, módulos o flujos que parezcan pesados
- renders innecesarios
- efectos mal planteados
- listeners/eventos no limpiados
- cálculos repetidos
- archivos muy grandes
- dependencias innecesarias o demasiado costosas
- código o estructura que pueda afectar la fluidez del sistema
- posibles cuellos de botella obvios

MANTENIBILIDAD / ARQUITECTURA
- código muy acoplado
- responsabilidades mezcladas
- módulos difíciles de testear o extender
- archivos demasiado largos
- naming inconsistente
- patrones mezclados
- duplicación de lógica
- deuda técnica visible
- decisiones arquitectónicas que hoy estén frenando velocidad de desarrollo

CÓDIGO MUERTO / BASURA TÉCNICA
- archivos aparentemente no usados
- funciones no usadas
- imports no usados
- ramas lógicas obsoletas
- mocks viejos
- flags o configuraciones heredadas
- código comentado que ya no aporta
- cualquier resto técnico que ensucie el proyecto o baje claridad

FORMATO DE ENTREGA OBLIGATORIO

Quiero que respondas de forma estructurada, clara y ejecutiva, usando este formato:

1. RESUMEN EJECUTIVO

- principales oportunidades
- principales riesgos

2. SCORE GENERAL DEL PROYECTO

- Seguridad: XX%
- Performance: XX%

- Escalabilidad: XX%
- Limpieza técnica: XX%

IMPORTANTE:
- No inventes números vacíos.

3. FORTALEZAS DETECTADAS
- listado de cosas bien resueltas
- justificar por qué son fortalezas
- explicar cómo ayudan al producto o al desarrollo

4. DEBILIDADES DETECTADAS
- listado de problemas o fragilidades
- justificar por qué son debilidades reales
- indicar impacto probable

5. RIESGOS PRINCIPALES
Separá por prioridad:
- riesgo alto
- riesgo medio

Para cada riesgo indicá:

- basura técnica visible

7. CUELLOS DE BOTELLA Y PROBLEMAS DE FLUIDEZ
- qué cosas pueden estar afectando experiencia de uso o performance

8. RECOMENDACIONES PRIORIZADAS
Necesito una hoja de ruta corta, ordenada por impacto:
- Quick wins
- Mejoras medianas
- Mejoras estructurales

Para cada recomendación:
- qué conviene hacer
- por qué
- impacto estimado
- riesgo estimado
- complejidad estimada

9. TOP 5 CAMBIOS RECOMENDADOS
- dame los 5 cambios más valiosos para hacer primero
- ordenados por retorno técnico/producto

- nombrá los archivos, carpetas o módulos más relevantes que tomaste en cuenta para formar criterio
