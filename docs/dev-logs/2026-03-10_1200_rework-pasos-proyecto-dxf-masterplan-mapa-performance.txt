── DIAGNÓSTICO TÉCNICO ──────────────────────────────────────────────────────────

### BUG 1 — CRÍTICO: Mapa completamente roto (Leaflet comentado)
Archivo: components/masterplan/masterplan-map.tsx
Causa: Las 3 importaciones de Leaflet estaban comentadas y reemplazadas con
       `const L: any = null;`. Esto hacía que cada intento de inicializar el mapa,
       renderizar el overlay y dibujar polígonos fallara silenciosamente (null.map()).
Fix: Descomentar los 3 imports de Leaflet dynamic.

### BUG 2 — Sync requería ADMIN (desarrolladores no podían sincronizar)
Archivo: app/api/proyectos/[id]/blueprint/sync/route.ts
Causa: La verificación era `session.user.role !== "ADMIN"` sin verificar propiedad.
Fix: Ahora permite ADMIN (acceso total) o cualquier usuario que sea dueño del
     proyecto (creadoPorId === session.user.id).

### BUG 3 — Pérdida de estado al cambiar de tab en Paso 2 (blueprint)
Archivo: components/masterplan/blueprint-engine.tsx
Causa: El estado (svgContent, extractedPaths, lotRecords) es local (useState).
       Al cambiar de tab, el componente se desmonta → todo se pierde.
Fix: Se agrega useEffect al montar que llama al nuevo GET /api/proyectos/[id]/blueprint.
     Si existe masterplanSVG + unidades sincronizadas, reconstruye el estado desde la BD.
     Badge "Plano guardado" indica que los datos vienen del servidor.

### BUG 4 — ViewBox hardcodeado en masterplan-viewer
Archivo: components/masterplan/masterplan-viewer.tsx
Causa: `<svg viewBox="0 0 1000 800">` fijo. Las geometrías del DXF tienen coordenadas
       en sistemas de dibujo que pueden ser 0-500, 0-2000, etc.
       Resultado: los lotes se renderizaban fuera de la vista.
Fix: useMemo calcula el viewBox dinámicamente desde las coordenadas reales de los
     paths de las unidades. Padding de 6% del bounding box.

### PROBLEMA 5 — Numeración de lotes "140 y pico"
Causa: El DXF tiene textos "140", "141"... que el parser detecta como `lotNumber`.
       El sistema los guarda como `numero` en la BD y los muestra tal cual.
       El `internalId` (1, 2, 3...) existe pero solo se veía en la columna "#".
Fix:
  - En coordenadasMasterplan se guarda ahora también `lotLabel` (texto del DXF)
  - En masterplan-viewer, el label del polígono muestra `internalId` si está disponible
    (orden espacial 1, 2, 3...) en lugar del texto del DXF
  - En blueprint-engine, la columna "#" muestra el internalId y "N° Lote" el texto DXF
  - Claramente separados: internalId = posición lógica, numero = texto del DXF

── INVESTIGACIÓN GOOGLE MAPS ────────────────────────────────────────────────────

  Estado actual: Se usa Leaflet con Google tile layers (sin API key oficial).
    - Satellite: https://mt1.google.com/vt/lyrs=s&x={x}&y={y}&z={z}
    - Streets:   https://mt1.google.com/vt/lyrs=m&x={x}&y={y}&z={z}
    - Hybrid:    https://mt1.google.com/vt/lyrs=y&x={x}&y={y}&z={z}

  Análisis de alternativas:

  OPCIÓN A (actual): Leaflet + Google tiles sin API key
    ✓ Funciona hoy sin costo
    ✓ Hasta zoom 21
    ✗ Términos de servicio de Google no lo permiten explícitamente
    ✗ Sin acceso a Geocoding, Places, Directions
    ✗ Sin soporte oficial

  OPCIÓN B: Google Maps JavaScript API oficial
    APIs necesarias: Maps JavaScript API + (opcional) Geocoding API
    Costo: ~$0.007 por 1000 cargas de mapa, $5/1000 geocoding requests
    Free tier: $200/mes crédito Google → ~28,000 cargas de mapa gratis
    Lib: @react-google-maps/api o @vis.gl/react-google-maps
    ✓ Términos claros, soporte oficial
    ✓ Overlay de imágenes nativo (GroundOverlay)
    ✓ Geocoding, Street View, satellite HD
    ✗ Requiere API key + billing activado
    Pendiente: requiere GOOGLE_MAPS_API_KEY en .env

  OPCIÓN C: MapboxGL JS
    ✓ Open source, mejor performance para polígonos grandes
    ✓ Soporte nativo para GeoJSON overlays y SVG
    ✗ Tiles de satelite requieren cuenta Mapbox (free tier 50k cargas/mes)
    ✗ Cambio de biblioteca (mayor refactor)

  RECOMENDACIÓN: Continuar con Leaflet (opción A) en desarrollo local y demo.
  Para producción, migrar a Google Maps JS API (opción B) con API key.
  No se requiere migrar la arquitectura — Leaflet y Google Maps tienen APIs similares.
